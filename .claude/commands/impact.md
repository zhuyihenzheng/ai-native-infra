# /impact — 影响调查

对 `$ARGUMENTS` 做改动前影响调查。这是修改既存代码的**前置义务**（见 change-safety.md）。

## 步骤

1. 若输入是式样书路径：先读式样书，列出它涉及的类/表/画面/API，逐个作为调查对象。
2. 派 `code-analyst` 子代理执行调查（对象多时可并行分派），要求返回其标准报告格式。
3. 汇总为一份影响范围结论：
   - **必改**：直接引用清单（文件:行）
   - **需判断**：间接影响（如共用 SQL 片段、共通部品、被复制的旧代码）
   - **连带**：受影响的测试、fixture、式样书、testcases
4. 明说调查盲区（动态拼接的类名/SQL、反射、配置驱动的调用等静态检索抓不到的）。

## 输出

- 调查结果输出到会话；正在做某案件时（存在 `ai/work/<ticket>/`），同时存为 `ai/work/<ticket>/impact.md`。
- 影响面超出用户描述的预期范围时，**先停下来报告**，不要直接开改。
