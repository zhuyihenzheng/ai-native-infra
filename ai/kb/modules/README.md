# modules/ — 模块笔记

每个业务模块一个 `<模块名>.md`，由 `/remember` 在该模块被实际开发/调查后写下。
接手涉及某模块的任务时，**先读它的笔记**（若存在）。

建议结构（保持一页以内，超了就删过时内容）：

```markdown
# <模块名>
> 最后更新: 日期 / 依据案件: ticket

## 入口与主流程        <!-- 关键类与调用链，带路径 -->
## 状态机 / 业务规则    <!-- 代码里不明显但必须知道的 -->
## 隐藏前提 / 慎动点    <!-- 「这里为什么这么怪」的答案 -->
## 关联表 / 关联模块
```
