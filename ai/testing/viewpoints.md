# 测试观点目录（観点カタログ）

> 测试设计时**逐项走查**本目录；不适用的观点显式标「対象外：理由」。防漏优先。
> 来源标记：`[DEF]` = 可从定义机械展开；`[SPEC]` = 需读式样书判断；`[REG]` = 既存改修回归专用。

## 1. 単項目（入力チェック）[DEF]
- 必須 / 任意（未入力・空白のみ・null）
- 型（数値項目に文字、日付項目に不正形式）
- 桁（上限ちょうど / 上限+1 / 0 桁）
- 文字種（全角半角、機種依存文字、サロゲートペア、前後空白）
- code 値（合法集内 / 集外 / 大文字小文字）

## 2. 相関チェック [SPEC]
- 項目間整合（開始日 ≤ 終了日、金額と区分の整合）
- 条件付き必須（A 選択時のみ B 必須）

## 3. 正常系バリエーション [SPEC]
- 代表値 / 検索 0 件・1 件・複数件・上限件数
- 分页（1 页 / 页境界 / 最終页 / 页超え指定）
- ソート（各列昇降順、白名单外列の拒否）

## 4. 異常系・業務エラー [SPEC]
- 式样书「異常系」表の全行
- 楽観ロック競合 / 排他制御
- 対象データ不存在（直後に他者削除）
- 外部 IF エラー（タイムアウト、業務エラー応答、システムエラー応答）

## 5. 状態・遷移 [SPEC]
- 状態機の全遷移（許可される遷移 / 禁止遷移の拒否）
- 画面遷移（正常 / 戻る / 二重送信 / 直接 URL）

## 6. 権限 [SPEC]
- 角色 × 操作の可否表全组合
- 権限なし时的行为（非表示 / 非活性 / エラー）；URL 直叩き

## 7. DB 整合 [DEF]
- 登録・更新後の列値（含更新者・更新日時等共通列）
- 削除の波及（論理削除フラグ / 関連表）
- トランザクション（途中失敗時のロールバック）

## 8. Mapper / SQL 層 [DEF]
- resultMap 映射正确性（NULL 列 → 字段）
- 動的 SQL 全分支（`<if>` 条件组合の代表 + 全 ON / 全 OFF）
- 分页 SQL の件数一致

## 9. 回帰（既存改修必須）[REG]
- 変更設計書 §5 の回帰観点全项
- 変更点を共有する共通部品の既存呼び出し元代表 1 条ずつ
- 変更していない動的 SQL 分支の挙動不変
