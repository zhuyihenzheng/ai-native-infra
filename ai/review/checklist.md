# Review Checklist

> `/review`（code-reviewer 代理）逐节走查；每节必须有结论（OK / 指摘#n）。
> 指摘必须有依据（规则条目 / examples 差异 / 缺陷逻辑），风格偏好不算指摘。

## 1. 对齐性
- [ ] 请求流未跳层；未新增项目没有的层
- [ ] 命名/注解/结构与 `ai/kb/examples/` 对应层样例一致
- [ ] 未引入项目未使用的依赖/技术栈

## 2. 禁止事项（core-contract forbidden + change-safety）
- [ ] 无依据未改：路由 / public 签名 / SQL 列 / 表结构 / 画面项目
- [ ] 无夹带改动：格式化、import 重排、重命名、无关重构
- [ ] diff 内文件都在実装方針的文件清单内（有方针时）

## 3. 正确性
- [ ] 空值/边界（null、空集合、0 件、上限）
- [ ] 事务边界正确；途中失败可回滚；无部分提交
- [ ] 异常未被吞没；错误消息走消息定义，不硬编码
- [ ] 并发/排他（楽観ロック处理与既存一致）

## 4. MyBatis / SQL
- [ ] 无 `${}` 拼接（白名单排序除外）
- [ ] 动态 SQL 改动未改变未涉及分支的语义
- [ ] 无循环内查询（N+1）；分页/排序与项目机制一致
- [ ] resultMap 与 DEF/DDL 一致

## 5. 测试
- [ ] 变更点有对应测试 case 与实装；回归观点已覆盖（改修案件）
- [ ] 测试方法嵌 traceability ID；断言主题单一；未为凑绿弱化断言
- [ ] fixture 全合成数据（无真实 PII / 凭证 / 客户数据 / 真实内部标识符）

## 6. 安全
- [ ] 输入校验在既定层完成；无注入面（SQL/路径/OS 命令）
- [ ] 无凭证/密钥硬编码；日志不输出机密与个人信息
- [ ] 权限检查未被绕过（新路由/新操作有与既存一致的権限制御）

## 7. 可追溯
- [ ] 新代码挂 traceability ID 回链式样书
- [ ] 改了式样书/DEF 时，同 ID 的 testcases/测试/fixture 已连带更新
- [ ] `ai/kb/lessons.md` 中的历史教训未重犯
