# 测试方针（通用，对齐时补项目细节）

## 两类来源
- **boundary**：来自 DEF，机械全覆盖（必須/型/桁/code 值/边界）。
- **behavior**：来自式样书，覆盖正常/异常/业务エラー/画面遷移。

## 分层观点（按本项目实际存在的层取用）
- 单元（业务层）：分支/边界/异常路径，依赖 mock。
- Mapper：SQL 正确性、resultMap 映射、动态 SQL 分支、分页。
- API/Client：请求映射、响应解析、业务/系统错误、mock 一致。
- 集成：跨层主流程 + 关键异常。
- 画面/E2E：导航 + 可见数据（涉及 UI 改动时为主验证，编译通过不算验证）。

## 纪律
- 全合成数据；如实报告红/绿；失败定位不顺手改无关代码。
- 测试方法名嵌 traceability ID。
